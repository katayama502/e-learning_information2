## 3. アーキテクチャ設計

### 3.1 全体アーキテクチャ

```
ブラウザ (React)
    │
    ├── Zustand Store (クライアント状態)
    │       ├── authStatus / activeRole
    │       ├── completedLessonIds (localStorage 永続化)
    │       └── courses / users
    │
    ├── Next.js App Router
    │       ├── Server Components (静的レンダリング)
    │       ├── Client Components ("use client")
    │       └── Server Actions (認証 / DB 書き込み)
    │
    └── API Routes (Route Handlers)
            │
            ▼
    Supabase (PostgreSQL)
            ├── courses
            ├── course_curriculums
            ├── course_lessons
            └── profiles
```

### 3.2 レイアウト階層

```
src/app/layout.tsx (ルートレイアウト)
└── <LayoutWrapper> (src/components/LayoutWrapper.tsx)
    │   ※ /reskill/** パスは LayoutWrapper のサイドバーをスキップ
    │
    ├── /reskill/** → src/app/reskill/layout.tsx (reskill 専用レイアウト)
    │       ├── デスクトップ: 左サイドバー (w-56)
    │       ├── モバイル: ハンバーガー → スライドドロワー
    │       └── /reskill/lesson/* → サイドバーなし（全画面プレイヤー）
    │
    ├── /admin/** → LayoutWrapper がスキップ（独自レイアウト）
    │
    └── その他ページ → LayoutWrapper のサイドバーを使用
```

**重要**: `isReskillPage` / `isAdminDashboard` / `isPublicPage` の判定により、
LayoutWrapper は各セクションのレイアウトに干渉しない設計になっています。

### 3.3 データフロー（学習者 UI）

```
1. ページロード
   reskill/page.tsx
   └── Promise.all([
         GET /api/elearning/tracks  → courses テーブル
         GET /api/elearning/tracks/[id]/courses → course_curriculums
       ])

2. コース一覧
   reskill/courses/page.tsx
   └── GET /api/elearning/modules?is_public=true

3. レッスン視聴
   reskill/lesson/[id]/page.tsx
   └── GET /api/elearning/content/[id]
       useAppStore.completeLesson(id) → Zustand persist

4. 進捗表示
   completedLessonIds (Zustand) と全レッスン数を比較
```

### 3.4 データフロー（管理者 UI）

```
admin/elearning/page.tsx
└── GET /api/elearning/tracks?showAll=true
    PUT/DELETE /api/elearning/tracks/[id]  (Service Role Key 使用)

admin/elearning/content/page.tsx
└── GET /api/elearning/modules  (全モジュール)
    POST /api/elearning/content (レッスン追加)
    POST /api/quiz/generate     (AI クイズ生成)
    ファイル読み込み → CSV パース → バルクインサート
```

## 4. コンポーネント設計

### 4.1 reskill/layout.tsx（reskill セクションレイアウト）

**責務**: `/reskill/**` 配下のすべてのページにサイドバーナビゲーションを提供

```tsx
// 主要な状態
const [isMenuOpen, setIsMenuOpen] = useState(false);  // モバイルドロワー開閉
const [isLoggingOut, setIsLoggingOut] = useState(false);

// 構成要素
├── <aside>           // デスクトップサイドバー (hidden md:flex)
│     ├── ロゴ
│     ├── <nav> renderNavLinks()
│     └── ログアウト/ログインボタン
├── <div> flex-1      // コンテンツエリア
│     ├── <header>    // モバイルヘッダー (md:hidden)
│     └── <main>      // ページコンテンツ {children}
├── <div> オーバーレイ // モバイルドロワー背景
└── <div> ドロワー    // モバイルスライドパネル
```

**ナビゲーション項目**:
- Dashboard (`/reskill`, `exact: true`)
- Courses (`/reskill/courses`)
- 管理者画面 (`/admin/elearning`) ※ isAdmin のみ

**アクティブ判定**:
- `exact: true` → `pathname === item.href`
- 通常 → `pathname.startsWith(item.href)`

### 4.2 LayoutWrapper.tsx（グローバルレイアウト）

**責務**: reskill/admin/public ページ**以外**にサイドバーを提供

```tsx
// バイパス条件
if (isPublicPage || isAdminDashboard || isReskillPage) {
    return <ErrorBoundary>{children}</ErrorBoundary>;
}
// → reskill ページはここで早期リターン、reskill/layout.tsx に委譲
```

### 4.3 Zustand Store（appStore.ts）

e-learning 関連の主要な状態:

| 状態 | 型 | 説明 |
|-----|---|------|
| `completedLessonIds` | `string[]` | 完了済みレッスン ID（localStorage 永続化） |
| `lastViewedLessonIds` | `string[]` | 最後に視聴した 10 件のレッスン ID |
| `isLessonSidebarOpen` | `boolean` | レッスンページのサイドバー開閉状態 |
| `isFetchingCourses` | `boolean` | コース取得中フラグ |

主要なアクション:
- `completeLesson(lessonId)` — レッスン完了記録
- `updateLastViewedLesson(lessonId)` — 最終視聴更新
- `fetchCourses()` — コース一覧 API 取得

## 5. Supabase クライアント設計

### 5.1 client.ts（ブラウザ用）

```typescript
// シングルトンパターン
let client: SupabaseClient | null = null;

export function createClient() {
    const url = process.env.NEXT_PUBLIC_SUPABASE_URL;
    const key = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY;
    if (client && url && key) return client;
    client = createBrowserClient(
        url ?? 'https://placeholder.supabase.co',  // ビルド時クラッシュ防止
        key ?? 'placeholder-key'
    );
    return client;
}
```

**注意**: プレースホルダー値はビルド時の prerender クラッシュ回避のため。実際の認証は環境変数が必要。

### 5.2 server.ts（サーバーコンポーネント / Server Actions 用）

`@supabase/ssr` の `createServerClient()` を使用し、Next.js の `cookies()` から Cookie を読み書き。

### 5.3 API Route 用クライアント

管理者操作（書き込み）には `SUPABASE_SERVICE_ROLE_KEY` を使用し、RLS をバイパス:

```typescript
const supabase = createClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.SUPABASE_SERVICE_ROLE_KEY || process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
);
```

## 6. 認証設計

### 6.1 認証フロー詳細

```
ブラウザ                    Server Action              Supabase Auth
   │                             │                         │
   ├─ POST /login ──────────────>│                         │
   │  { email, password }        │                         │
   │                             ├── signInWithPassword() >│
   │                             │                         │
   │                             │<── { session, user } ───┤
   │                             │                         │
   │                             ├── cookies().set(        │
   │                             │     'sb-*-auth-token',  │
   │                             │     session             │
   │                             │   )                     │
   │<── redirect('/reskill') ────┤                         │
```

### 6.2 セッション検証

`LayoutWrapper.tsx` で `window.focus` / `visibilitychange` イベントをトリガーにセッション再検証を実施。セッション切れの場合は `/login` へリダイレクト。

### 6.3 ログアウトフロー

1. `localStorage.removeItem('eis-app-store-v3')` — Zustand persist データ削除
2. `supabase.auth.signOut()` — クライアントサイドセッション削除
3. Server Action `logoutAction()` — HttpOnly Cookie 削除
4. `window.location.href = '/login'` — ページ遷移

## 8. セキュリティ設計

### 8.1 認証・認可の多層防御

| レイヤー | 実装 | 用途 |
|---------|------|------|
| L1: ミドルウェア | `utils/supabase/middleware.ts` | `/admin/*` アクセス制御 |
| L2: Supabase RLS | 各テーブルのポリシー | DB レベルのアクセス制御 |
| L3: UI 制御 | `isAdmin` フラグによる表示制御 | 管理者メニューの非表示 |

### 8.2 API セキュリティ

- 管理者 API は Service Role Key を使用（RLS バイパスのため内部のみ使用）
- 公開 API は `is_published = true` / `is_public = true` でフィルタリング

## 9. 既知の設計上の注意点

| 項目 | 内容 |
|-----|------|
| 進捗データの保存先 | `completedLessonIds` は Zustand + localStorage のみ。DB に保存されないため、別端末・ブラウザ間で同期されない |
| AI クイズ機能 | Anthropic API キーが未設定の場合は動作しない |
| YouTube 依存 | 動画コンテンツは全て YouTube 埋め込みのため、YouTube のサービス状態に依存する |
| ゲスト視聴 | 認証なしでコース・レッスンを閲覧可能（進捗は保存されない） |
