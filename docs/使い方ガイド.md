## 学習者向けガイド

### ログイン

1. ブラウザで `https://[サイトURL]/login` にアクセス
2. メールアドレスとパスワードを入力
3. 「ログイン」ボタンをクリック
4. ダッシュボード（`/reskill`）に自動遷移します

> **ゲスト利用**: ログインせずにコース・レッスンを閲覧することもできますが、学習進捗は保存されません。

---

### ダッシュボードの使い方

ログイン後に表示されるトップページです。

**表示内容:**

| 要素 | 説明 |
|-----|------|
| 学習トラック一覧 | DX・AI・ChatGPT 等のカテゴリ別トラック |
| 進捗バー | 各トラックの学習完了率（%）|
| 全体進捗 | 完了レッスン数 / 全レッスン数 |

**操作方法:**
- トラックカードをクリック → そのトラックのコース一覧へ
- 「続きから学習」ボタン → 最後に視聴したレッスンへ直接移動

---

### コースの受講

#### コース一覧を見る

1. 左サイドバー（PCの場合）または右上ハンバーガーメニュー（スマートフォンの場合）から「Courses」をクリック
2. 公開されているすべてのコースが一覧表示されます

#### コースを選んで学習を開始する

1. コース一覧からコースカードをクリック
2. コース詳細ページでモジュール構成を確認
3. 視聴したいレッスンをクリック → レッスン視聴ページへ

---

### レッスンの視聴

#### レッスン視聴ページの構成

```
┌────────────────────────────────────┐
│  ← 戻る  コース名 > レッスン名     │  ← ヘッダー
├──────────────┬─────────────────────┤
│              │                     │
│ レッスン     │  YouTube プレイヤー  │
│ 一覧         │                     │
│ (サイドバー) │                     │
│              │                     │
│ ✓ レッスン1 │  タイトル・説明文    │
│ > レッスン2 │                     │
│   レッスン3 │  [完了にする] ボタン │
└──────────────┴─────────────────────┘
```

> スマートフォンでは縦並びで表示されます。

#### レッスンを完了にする

動画を視聴したら「完了にする」ボタンをクリック。

- ✅ チェックマークが付きます
- 左サイドバーのレッスン一覧にも完了済みアイコンが表示されます
- ダッシュボードの進捗バーが更新されます

> **注意**: 進捗はブラウザのローカルストレージに保存されます。別のブラウザや端末では同期されません。

#### 前後のレッスンへ移動

- レッスンページ下部の「前のレッスン」「次のレッスン」ボタン
- または左サイドバーのレッスン一覧をクリック

---

### ナビゲーション

#### PC の場合

左サイドバーに常時表示されます。

| メニュー項目 | 遷移先 |
|------------|-------|
| Dashboard | /reskill（ダッシュボード） |
| Courses | /reskill/courses（コース一覧） |

#### スマートフォンの場合

1. 画面左上のハンバーガーアイコン（☰）をタップ
2. 左からスライドしてメニューが開きます
3. メニュー項目をタップして遷移
4. 画面外をタップするか×ボタンでメニューを閉じる

---

左サイドバー（PC）またはハンバーガーメニュー内（スマートフォン）の「ログアウト」ボタンをクリック。

## 管理者向けガイド

> 管理者画面は `user_type = 'admin'` のアカウントでのみアクセスできます。

### 管理者画面へのアクセス

1. 管理者アカウントでログイン
2. サイドバーに「管理者画面」が表示されます
3. クリックすると `/admin/elearning` へ遷移

---

### トラック管理

**アクセス**: `/admin/elearning`

学習トラック（DX入門・ChatGPT等のカテゴリ）を管理します。

#### トラックを作成する

1. 「トラックを追加」ボタンをクリック
2. 以下の情報を入力:
   - **タイトル**: トラック名（例: 「DX入門」）
   - **説明**: トラックの概要
   - **サムネイル画像**: 画像 URL
3. 「保存」ボタンをクリック

#### トラックを編集する

1. トラック一覧から対象トラックの「編集」ボタンをクリック
2. 情報を更新して「保存」

#### トラックを公開/非公開にする

- トラックの「公開」トグルスイッチで切り替え
- 非公開 → 学習者のダッシュボードに表示されない

#### トラックを削除する

- 「削除」ボタンをクリック → 確認ダイアログで「削除」

> **注意**: トラックを削除しても、紐付いているモジュール・レッスンは削除されません。

---

### コンテンツ管理（モジュール・レッスン）

**アクセス**: `/admin/elearning/content`

#### モジュール（コース）を作成する

1. 「モジュールを追加」ボタンをクリック
2. 入力:
   - **タイトル**: モジュール名
   - **所属トラック**: プルダウンでトラックを選択
   - **説明**: モジュールの概要
   - **公開設定**: 公開 / 非公開
3. 「保存」

#### レッスンを追加する

1. 対象モジュールを展開
2. 「レッスンを追加」ボタン
3. 入力:
   - **タイトル**: レッスン名
   - **YouTube URL**: 動画の URL（例: `https://www.youtube.com/watch?v=XXXXXX`）
   - **説明**: レッスン内容の説明
   - **所要時間**: 分単位で入力
4. 「保存」

#### YouTube URL の入力形式

以下のどの形式でも対応しています:
- `https://www.youtube.com/watch?v=VIDEO_ID`
- `https://youtu.be/VIDEO_ID`

#### CSV でレッスンを一括インポートする

1. 「CSV インポート」ボタンをクリック
2. CSV ファイルを選択（または ドラッグ＆ドロップ）
3. CSV フォーマット:

```csv
title,youtube_url,description,duration_minutes,order_index
DXとは何か,https://youtu.be/XXXXX,DX の基礎を解説します,10,1
デジタル化の進め方,https://youtu.be/YYYYY,実践的な進め方,15,2
```

4. プレビューを確認して「インポート実行」

#### AI クイズを自動生成する

1. レッスン一覧から「クイズ生成」ボタンをクリック
2. AI（Claude）がレッスン内容に基づいてクイズを自動生成
3. 生成されたクイズを確認・編集して保存

> **前提**: `ANTHROPIC_API_KEY` 環境変数が設定されている必要があります。

## デプロイ・環境構築ガイド

### ローカル開発環境のセットアップ

#### 前提条件

- Node.js 20 以上
- npm 10 以上
- Supabase アカウント

#### 手順

```bash
# 1. リポジトリをクローン
git clone [repository-url]
cd e-learning

# 2. 依存関係をインストール
npm install

# 3. 環境変数を設定
cp .env.example .env.local
# .env.local を編集して以下を設定:
# NEXT_PUBLIC_SUPABASE_URL=https://xxxxxxxxxx.supabase.co
# NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci...
# SUPABASE_SERVICE_ROLE_KEY=eyJhbGci...
# ANTHROPIC_API_KEY=sk-ant-... (任意)

# 4. 開発サーバーを起動
npm run dev
```

ブラウザで `http://localhost:3000` を開いてください。

---

### データベースの初期セットアップ

#### Supabase プロジェクトの作成

1. [supabase.com](https://supabase.com) でプロジェクトを作成
2. `Project URL` と `anon key` と `service_role key` を取得
3. `.env.local` に設定

#### テーブルの作成

Supabase ダッシュボードの「SQL Editor」で以下を実行（スキーマが未作成の場合）:

```sql
-- courses テーブル
CREATE TABLE courses (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title TEXT NOT NULL UNIQUE,
    description TEXT,
    image TEXT,
    category TEXT DEFAULT 'Track',
    is_published BOOLEAN DEFAULT false,
    order_index INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- course_curriculums テーブル
CREATE TABLE course_curriculums (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    course_id UUID REFERENCES courses(id),
    title TEXT NOT NULL,
    description TEXT,
    is_public BOOLEAN DEFAULT false,
    status TEXT DEFAULT 'pending',
    order_index INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- course_lessons テーブル
CREATE TABLE course_lessons (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    curriculum_id UUID REFERENCES course_curriculums(id),
    title TEXT NOT NULL,
    youtube_url TEXT,
    description TEXT,
    duration_minutes INTEGER,
    order_index INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- profiles テーブル（Supabase Auth と連携）
CREATE TABLE profiles (
    id UUID PRIMARY KEY REFERENCES auth.users(id),
    user_type TEXT DEFAULT 'seeker',
    full_name TEXT,
    email TEXT,
    avatar_url TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### 初期コンテンツのシード

```bash
# Supabase SQL Editor でスクリプトを実行
# または psql コマンドで:
psql [connection-string] < scripts/seed_ai_content.sql
```

#### 管理者アカウントの作成

1. Supabase ダッシュボード → Authentication → Users
2. 「Add user」で管理者のメール・パスワードを作成
3. 作成されたユーザーの ID をコピー
4. SQL Editor で実行:

```sql
UPDATE profiles SET user_type = 'admin' WHERE id = '[ユーザーID]';
```

### Netlify へのデプロイ

#### 1. Netlify サイトを作成

1. [app.netlify.com](https://app.netlify.com) にログイン
2. 「Add new site」→「Import an existing project」
3. GitHub リポジトリを選択

#### 2. ビルド設定

Netlify のサイト設定 → Build & deploy:

```
Build command: npm run build
Publish directory: .next
```

> `netlify.toml` が存在するため自動で読み込まれます。

#### 3. 環境変数の設定

Netlify サイト設定 → Environment variables で追加:

| 変数名 | 値 |
|-------|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase プロジェクト URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase 匿名キー |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase サービスロールキー |
| `ANTHROPIC_API_KEY` | Anthropic API キー（任意） |

> **注意**: `EXT_PUBLIC_SUPABASE_ANON_KEY` ではなく `NEXT_PUBLIC_SUPABASE_ANON_KEY` が正しいキー名です（`N` が必要）。

#### 4. デプロイ実行

GitHub にプッシュすると自動的にデプロイが開始されます。

手動でデプロイする場合:
- Netlify ダッシュボード → Deploys → 「Trigger deploy」

## トラブルシューティング

### ログインできない

**症状**: ログインボタンを押しても何も起きない、エラーが表示される

**確認事項**:
1. メールアドレスとパスワードが正しいか確認
2. Supabase の Authentication → Users でユーザーが存在するか確認
3. ブラウザのコンソールでエラーメッセージを確認
4. `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` 環境変数が正しく設定されているか確認

---

### 管理者画面にアクセスできない

**症状**: `/admin/elearning` にアクセスすると `/login` にリダイレクトされる

**確認事項**:
1. 管理者アカウントでログインしているか確認
2. Supabase の `profiles` テーブルで `user_type = 'admin'` が設定されているか確認:
   ```sql
   SELECT id, user_type FROM profiles WHERE email = 'あなたのメール';
   ```
3. `user_type` が `'admin'` でなければ更新:
   ```sql
   UPDATE profiles SET user_type = 'admin' WHERE email = 'あなたのメール';
   ```

---

### コースが表示されない

**症状**: ダッシュボードやコース一覧が空で表示される

**確認事項**:
1. `courses` テーブルに `is_published = true` のデータが存在するか確認
2. `course_curriculums` テーブルに `is_public = true` のデータが存在するか確認
3. 初期データ（`scripts/seed_ai_content.sql`）を実行済みか確認

---

### Netlify ビルドが失敗する

**症状**: Netlify のビルドログに `Next.js build worker exited with code: 1` が表示される

**よくある原因と対処法**:

| 原因 | 対処法 |
|-----|--------|
| 環境変数が未設定 | Netlify の Environment variables で設定 |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` のタイポ | キー名の先頭が `NEXT_PUBLIC_` になっているか確認（`EXT_PUBLIC_` ではない） |
| Node.js バージョン不一致 | `netlify.toml` で `NODE_VERSION = "20"` を設定 |
| `@netlify/plugin-nextjs` 未設定 | `netlify.toml` に `[[plugins]] package = "@netlify/plugin-nextjs"` を追加 |

---

### 学習進捗がリセットされた

**症状**: 完了済みレッスンのチェックマークが消えた

**原因**: 進捗はブラウザのローカルストレージに保存されているため、以下の場合にリセットされます:
- ブラウザのキャッシュ・サイトデータをクリアした
- 別のブラウザを使用した
- プライベート/シークレットモードで閲覧した
- 別の端末でアクセスした

**対処法**: 現在のバージョンでは端末間同期はサポートされていません。同じブラウザ・端末で継続して学習してください。

---

### YouTube 動画が再生されない

**症状**: レッスンページで動画が表示されない、エラーが表示される

**確認事項**:
1. インターネット接続を確認
2. YouTube へのアクセスがブロックされていないか確認（学校・職場のネットワーク等）
3. 動画 URL が正しいか管理者に確認
4. 動画が削除・非公開になっていないか YouTube で直接確認
